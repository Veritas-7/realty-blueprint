### Critical Finding
`SEOHead` is currently only rendered in `NotFound.tsx`. No other page renders it, meaning route-level meta tags (title, og, canonical, robots, JSON-LD) are NOT applied on any page except 404. This is the top-priority fix.

---

### 1. SEOHead Global Integration + JSON-LD Enhancement

**Move SEOHead into `AppLayout.tsx`** so it runs on every route automatically. Remove `is404` prop pattern — instead, SEOHead auto-detects missing routeMeta and falls back to `notFoundMeta` when path isn't in `routeMeta`.

**Enhance `SEOHead.tsx`:**
- Auto-render based on `routeMeta[path]` existence (fallback to notFoundMeta if not found AND we're on 404 route)
- Use `jsonLdType` from routeMeta to generate appropriate JSON-LD beyond just BreadcrumbList/WebSite
- Add `RealEstateAgent` JSON-LD for `/industry-overview`
- Remove `<SEOHead is404 />` from `NotFound.tsx` — instead, NotFound sets a flag or SEOHead detects `*` route

**Modify `src/data/seo-config.ts`:**
- Add `RealEstateAgent` JSON-LD generator function
- Make `generateJsonLd(path, meta)` that reads `jsonLdType` array and produces the right JSON-LD objects

### 5. SEOHead NotFound Handling

**Modify `NotFound.tsx`:** Remove `<SEOHead is404 />`. Instead, `SEOHead` in `AppLayout` will check if current path has no `routeMeta` entry — if so, apply `notFoundMeta`.

But there's a problem: AppLayout wraps ALL routes including valid ones AND the `*` catch-all. The solution:
- SEOHead checks `routeMeta[path]` — if missing, uses `notFoundMeta`
- This naturally handles 404 since the `*` route renders NotFound, and `path` won't be in `routeMeta`
